### Проектирование ER диаграммы 


Хранилище данных для банка

**Основные сущности:** клиенты, счета, транзакции, кредиты, платежи по кредитам, сотрудники

**Атрибуты:**  
Клиенты: ID клиента, Имя, Фамилия, Дата рождения, Адрес, Телефон, Тип клиента (физическое/юридическое лицо)  
Счета: ID счета, Тип счета (депозитный, кредитный), Баланс, Дата открытия, ID клиента  
Транзакции: ID транзакции, Сумма, Дата, Тип транзакции (пополнение, снятие, перевод), ID счета отправителя, ID счета получателя  
Кредиты: ID кредита, Сумма кредита, Процентная ставка, Срок кредита, ID клиента  
Платежи по кредитам: ID платежа, Сумма платежа, Дата платежа, ID кредита  

**Взаимосвязи:**  
Клиенты — Счета (1 ко многим)  
Счета — Транзакции (1 ко многим)  
Клиенты — Кредиты (1 ко многим)  
Кредиты — Платежи по кредитам (1 ко многим)  

**Бизнес-правила:**  
Клиент может иметь несколько активных счетов разных типов.  
Транзакции между счетами должны учитывать лимиты и типы счетов.  
Платежи по кредитам привязаны к конкретному кредиту, и их история должна быть доступна для аудита.
